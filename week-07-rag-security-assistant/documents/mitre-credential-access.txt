This document contains MITRE ATT&CK techniques related to Credential Access.
These techniques describe how attackers obtain account credentials to escalate privileges and maintain persistence.

Technique: Brute Force (T1110)
Description:
Brute force attacks involve systematically attempting many password combinations until the correct one is found. This method is commonly used against login portals, SSH services, and remote desktop interfaces.

Procedure Examples:
Attackers use automated tools to perform password spraying (trying common passwords across many accounts) or credential stuffing (using leaked credentials from previous breaches). Successful attempts allow attackers to gain unauthorized access to systems.

Technique: Credential Dumping (T1003)
Description:
Credential dumping involves extracting plaintext passwords, hashes, or Kerberos tickets from memory or local storage on compromised systems.

Procedure Examples:
Attackers often use tools such as Mimikatz to dump credentials from the LSASS process in Windows systems. These credentials can then be reused to move laterally or escalate privileges within the network.

Technique: OS Credential Dumping
Description:
Attackers target operating system mechanisms that store credentials, such as the Security Account Manager (SAM) database or cached domain credentials.

Procedure Examples:
By accessing system files or registry hives, attackers can extract hashed passwords and attempt offline cracking. They may also retrieve cached credentials stored for offline logins and reuse them across systems.
